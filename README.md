# Dragon

A cross-platform Flutter application with Firebase Authentication, themed with the [Catppuccin Mocha](https://catppuccin.com) color palette.

## Tech Stack

- **Flutter** — cross-platform UI (Android, Web, Windows, Linux)
- **Firebase Auth** — email/password authentication
- **Cloud Firestore** — database *(future)*
- **go_router** — declarative routing with auth-state redirect

---

## Prerequisites

Make sure you have the following installed before cloning:

- [Flutter SDK](https://docs.flutter.dev/get-started/install) (3.x or later)
- [Firebase CLI](https://firebase.google.com/docs/cli#install_the_firebase_cli)
- [FlutterFire CLI](https://firebase.flutter.dev/docs/cli/)

```bash
# Install Firebase CLI (if not already)
npm install -g firebase-tools

# Install FlutterFire CLI
dart pub global activate flutterfire_cli
```

---

## Setup After Cloning

### 1. Clone the repository

```bash
git clone <repo-url>
cd dragon
```

### 2. Install Flutter dependencies

```bash
flutter pub get
```

### 3. Connect Firebase

> `firebase_options.dart` and `google-services.json` are gitignored — you must generate them yourself.

Log in to Firebase, then run FlutterFire configure inside the project folder:

```bash
firebase login
flutterfire configure --platforms=android,web,windows
```

Select your Firebase project when prompted. This will generate:
- `lib/firebase_options.dart`
- `android/app/google-services.json`

### 4. Enable Email/Password sign-in

In the [Firebase Console](https://console.firebase.google.com):

1. Open your project
2. Go to **Authentication → Sign-in method**
3. Enable **Email/Password**

### 5. Run the app

**Web:**
```bash
flutter run -d web-server --web-port=8080
# Then open http://localhost:8080 in your browser
```

**Android** (emulator or physical device):
```bash
flutter run -d android
```

**Windows:**
```bash
flutter run -d windows
```

## Notes

- `lib/firebase_options.dart` is gitignored — every developer must run `flutterfire configure` after cloning.
- `ios/` and `macos/` platform folders are present but Firebase is **not configured** for them (requires Xcode).
